# Change Report

Use this structure when reporting completed engineering work:

```text
Implemented <short behavior>.

Changed:
- <file or module>: <what changed>

Verified:
- <command>: <result>

Not verified:
- <check>: <reason>

Notes:
- <assumption, follow-up, or unrelated issue noticed>
```

Keep it short. Do not claim checks that were not run.
