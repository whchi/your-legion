# Engineering Guardrails

- The repository is the system of record. Decisions that affect future agents should be saved in repo docs.
- Prefer local conventions over new abstractions.
- Keep diffs traceable to the request.
- Do not register Your Legion domain skills as harness top-level skills.
- Use tests to protect behavior, not implementation trivia.
- Treat destructive operations, data rewrites, force pushes, and broad dependency upgrades as approval-gated work.

When a task conflicts with these guardrails, name the conflict before changing behavior.
